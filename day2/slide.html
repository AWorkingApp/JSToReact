Single Page Application

Single-Page Applications (SPAs) are Web apps that load a single HTML page and dynamically update that page as the user interacts with the app. SPAs use AJAX and HTML5 to create fluid and responsive Web apps, without constant page reloads.

SPA Advantages

Let's focus on the good part •ᴗ•

  • Highly Responsive/Better User Experience
  • Super Fast and easy to scale
  • Easy Production Deployment

What is ReactJS

React is a declarative, efficient, and flexible JavaScript library for building user interfaces.
  • It uses a Virtual DOM
  • A Javascript library, not a framework
  • For building user interface
  • Component based

About Virtual DOM

In React, for every DOM object, there is a corresponding "virtual DOM object." A virtual DOM object is a representation of a DOM object, like a lightweight copy. ... Manipulating the DOM is slow. Manipulating the virtual DOM is much faster, because nothing gets drawn onscreen.

Why Virtual DOM

  • Browser DOM slow to manipulate
  • Browser DOM is so large and not efficient

Virtual DOM vs DOM

About Component

Components let you split the UI into independent, reusable pieces, and think about each piece in isolation.

About Component

  • Components are just like javascript functions
  • It has inputs, called props
  • It has internal states, called state
  • Functional Component
  • Class Component

State in Component

  • Used as Component's internal state
  • Initialized in constructor or as static property
  • Use this.setState() to update state, do not modify directly
  • Update state will trigger a rerender
  • State Update are merged, you only update what you need

Component Lifecycle

Every Class Component has its own lifecycle hooks

For Example: `componentWillMount()` is called before render and `componentWillUnmount()` is called after component is removed from DOM.
