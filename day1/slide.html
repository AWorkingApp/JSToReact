this in Javascript

In most cases, the value of this is determined by how a function is called. It can't be set by assignment during execution, and it may be different each time the function is called. ES5 introduced the bind method to set the value of a function's this regardless of how it's called, and ES2015 introduced arrow functions whose this is lexically scoped (it is set to the this value of the enclosing execution context).

So what does this mean???

Figure out this

                        
function printName(){ console.log(this.name); }

let mike = { name: 'mike', printName: printName }

let jim = { name: 'jim' }

printName(); // output: undefined
mike.printName(); // output: mkie
// jim.printName(); // not a function

printName = printName.bind(jim); // bind printName with jim
printName(); // output: jim

Arrow Function

An arrow function expression has a shorter syntax than a function expression and does not bind its own this, arguments, super, or new.target.
                        
                            //Example arrow function

                            let hw = () => { console.log("hello world") };
                            hw(); // output: hello world

() => {this}

                        
function printName(name) { console.log(name); }

let mike = { 
    name: 'mike', 
    printName: () => { console.log(this.name); },
    print: () => { printName(this.name); },
    printFunc: function () { printName(this.name); }
}
mike.printName(); // undefined
mike.print(); // undefined
mike.printFunc(); // mike

Closure

What is closure?

A closure is the combination of a function and the lexical environment within which that function was declared.

Closure

                        
let f = function (name){
    return function (){
        console.log(name);
    }
}

let func = f('test'); 
func();// output test

Immediately-Invoked Function Expression (IIFE)

Functions in JavaScript are first class objects. This means that JavaScript functions are just a special type of object that can do all the things that regular objects can do.

Immediately-Invoked Function Expression

                        
let f = function print(name){
    console.log(name);
}

let func = f('test'); 
func();// output test
f('test')(); // output test

High Order Function

In mathematics and computer science, a higher-order function (also functional, functional form or functor) is a function that does at least one of the following: takes one or more functions as arguments (i.e., procedural parameters), returns a function as its result.

High Order Function

                        
function calculate(x, y, cb){
    let sum = x + y;
    cb(sum)
}

calculate(1,2, function(sum){ console.log(`sum: ${sum}`);})

Curry Function

In mathematics and computer science, currying is the technique of translating the evaluation of a function that takes multiple arguments (or a tuple of arguments) into evaluating a sequence of functions, each with a single argument.

Curry Function

                        
function add(x){
    return function (y) {
        return x+y;
    }
}

console.log(add(1)(2)); // output 3
